# RepoForge

Take raw repos. Test them. Shape them. Ship them.

RepoForge is a structured pipeline for evaluating, integrating, documenting, and branding external tools and repositories. One prompt in - full security audit, trial integration, smoke test, graded scorecard, human SOP, AI agent context doc, and a branded GitHub fork out.

## The Problem

Integrating a new tool into your stack is a multi-step process that usually looks like: skim the README, install it, hit some errors, debug for a while, get it working, forget to document it, and then six months later nobody remembers how it works or why it's there.

RepoForge turns that into a repeatable 7-phase pipeline with structured artifacts at every step.

## Pipeline Phases

```
Repo URL dropped
    |
    v
Phase 1: Intake .............. Capture metadata, create project directory
    |
    v
Phase 2: Security Audit ...... Secrets scan, CVE check, permission scope, license
    |                          CRITICAL finding = STOP
    v
Phase 3: Trial Integration ... Install in sandbox, log every step
    |
    v
Phase 4: Smoke Test .......... Real-world scenarios, performance, error handling
    |
    v
Phase 5: Grade Card .......... Weighted rubric score, adopt/reject decision
    |
    v
Phase 6: Documentation ....... Human SOP + AI agent context doc
    |
    v
Phase 7: Fork & Brand ........ Fork, rebrand, strip, push to your org
```

## Grading Rubric

| Category | Weight |
|---|---|
| Security posture | 25% |
| Functionality | 25% |
| Integration effort | 15% |
| Maintenance health | 10% |
| Performance | 10% |
| Documentation quality | 10% |
| License compatibility | 5% |

**Score thresholds:**
- **4.0+** Full adopt - integrate as-is
- **3.0-3.9** Partial adopt - extract useful parts, build your own wrapper
- **2.0-2.9** Soft reject - document learnings, move on
- **Below 2.0** Hard reject

The rubric weights and thresholds are configurable in `rubric.json`.

## Quick Start

```bash
# Initialize a new evaluation
python3 forge.py init my-tool --repo https://github.com/org/tool --purpose "What this solves for us"

# Check status
python3 forge.py status my-tool

# After completing phases 2-4, create scores.json in the project directory:
# {"security": 4, "functionality": 5, "integration": 3, "maintenance": 4, "performance": 4, "docs": 3, "license": 5}

# Calculate weighted score and decision
python3 forge.py grade my-tool

# List all projects
python3 forge.py list
```

## Project Structure

```
repoforge/
  forge.py              # CLI orchestrator
  rubric.json           # Scoring weights and thresholds
  SKILL.md              # AI agent skill definition (OpenClaw/Claude)
  templates/
    security-report.md      # Security audit report template
    grade-card.md           # Grading rubric template
    sop-template.md         # Human SOP template
    agent-context-template.md  # AI agent context doc template
  docs/
    architecture.md         # Full pipeline design document
    example-sop.md          # Example SOP (PostHog LLM Analytics)
```

## Output Per Project

Each evaluation produces a structured project directory:

```
forge/<project-name>/
  intake.json           # Metadata, repo URL, purpose, status
  security-report.md    # Phase 2: security audit findings
  trial-log.md          # Phase 3: step-by-step integration record
  smoke-test-report.md  # Phase 4: test results
  scores.json           # Phase 5: raw category scores
  grade-card.md         # Phase 5: scored rubric with decision
  sop.md                # Phase 6: human-facing SOP
  agent-context.md      # Phase 6: AI agent context doc
  fork-manifest.json    # Phase 7: fork metadata
```

## AI Agent Integration

RepoForge includes a `SKILL.md` that allows AI agents (OpenClaw, Claude Code, or similar) to run the full pipeline autonomously. Drop a repo URL, the agent handles all 7 phases, and delivers the graded evaluation with documentation.

The `agent-context-template.md` generates structured docs that AI agents can consume - formatted for `INFRA.md` / `MEMORY.md` integration so agent knowledge stays current.

## Templates

All templates use `{{variable}}` placeholders. The agent fills these during Phase 6 based on context gathered in Phases 1-5. Templates are intentionally simple markdown - no build step, no templating engine dependency.
